# Contributing to EZPassRI


Thank you for your interest in contributing to EZPassRI! We welcome contributions of all kinds â€” bug reports, documentation, examples, tests, and code improvements.


## Contribution Guidelines
- Be respectful and professional.
- Follow the project's coding style and naming conventions.
- Keep changes focused and well-described.


## How to Contribute
1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature`.
3. Make your changes, add tests where appropriate.
4. Commit your changes with clear messages.
5. Push to your fork and open a pull request against `main`.


## Reporting Issues
- Search existing issues before opening a new one.
- Provide a clear title, steps to reproduce, expected vs actual behaviour, and environment details.


## Suggesting Features
- Open an issue describing the feature and use-case.
- Link related issues or PRs.


## Code of Conduct
Be kind, inclusive, and respectful. Any harassment or abusive behaviour will not be tolerated.


## Review Process
- Pull requests will be reviewed by maintainers.
- Reviews may request changes; please address feedback promptly.
- Once approved and CI passes, a maintainer will merge.
